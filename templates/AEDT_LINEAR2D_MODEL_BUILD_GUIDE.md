# AEDT 中一步步创建所需 2D 模型的中文教程

这份文档是专门给你“手工搭建 `Linearized2D` 模型本体”用的。

它和 `LINEAR2D_TEMPLATE_SETUP.md` 的区别是：

- `LINEAR2D_TEMPLATE_SETUP.md` 更偏模板契约和自动化准备
- 这份文档更偏“你在 AEDT 里实际怎么一步步把 2D 模型搭出来”

这份文档默认你的目标是：

- 先做一个稳定可求解的 2D 粗筛模型
- 不急着一上来完全拟真
- 先让自动化跑起来

## 1. 先明确这不是最终 3D 电机

你当前要做的 `Linearized2D`，不是最终真实的轴向磁通全结构。

第一版的推荐目标是：

- 建一个等效 2D 周期切片
- 让磁钢、气隙、线圈、背铁之间关系合理
- 让关键变量真的驱动模型变化
- 让 setup 和报表都能稳定导出

这一步只负责 2D 粗筛。

## 2. 最低可运行目标是什么

如果你想先最快跑通，第一版只要达到下面这个最低标准就行：

- 一个 `Linearized2D` 设计
- 一个 `Setup_2D`
- 一组能驱动几何的变量
- 一套能求解的 2D 几何
- 五个可导出报表

只要这五条齐了，就已经可以开始第一轮自动化。

## 3. 建模前先把变量准备好

你应该已经通过 bootstrap 脚本或手工方式建好了这些变量。

至少要有：

- `outer_diameter_mm`
- `inner_diameter_mm`
- `pole_count`
- `magnet_thickness_mm`
- `pole_arc_ratio`
- `airgap_mm`
- `backiron_thickness_mm`
- `coil_radial_span_mm`
- `coil_mean_radius_mm`
- `turns_per_phase`
- `conductor_width_mm`
- `conductor_thickness_mm`
- `parallel_strands`
- `phase_current_rms`
- `speed_rpm`
- `magnet_segments_per_pole`

建议 helper 变量也一起建好：

- `outer_radius_mm`
- `inner_radius_mm`
- `mean_radius_mm`
- `mean_diameter_mm`
- `pole_pitch_mm`
- `coil_inner_radius_mm`
- `coil_outer_radius_mm`
- `period_length_mm`
- `magnet_arc_mm`

如果你想让脚本先帮你搭一个“可复核的 2D 几何骨架”，现在也可以直接运行：

- `scripts/build_linear2d_model.py`

如果 AEDT host 已经在运行，也可以从工作区外部排队执行：

- `.\launchers\Queue-BuildLinear2DModel.ps1`

这个脚本会尝试自动创建：

- `Auto2D_BackIron`
- `Auto2D_Magnet_N`
- `Auto2D_Magnet_S`
- `Auto2D_AirGap`
- `Auto2D_Coil_Pos`
- `Auto2D_Coil_Neg`
- `Auto2D_Region`

它的定位是“先把参数化几何骨架搭出来”，不是一次性把所有磁化方向、激励和边界都做完。

## 4. 先决定 2D 画图坐标怎么理解

第一版推荐这样理解：

- 水平方向：圆周展开后的周期方向
- 竖直方向：磁路厚度方向

也就是：

- 左右表示一个极对内的位置变化
- 上下表示磁钢、气隙、线圈、背铁的堆叠

这样最适合做“线性展开的轴向磁通等效切片”。

## 5. 第一步：先画外围空气区域

建议你先不要急着画磁钢和线圈，先画整个模型的外框。

### 5.1 为什么先画空气区

因为这样可以先把：

- 模型尺度
- 周期长度
- 上下边界

这些最基础的东西确定下来。

### 5.2 建议怎么定大小

第一版建议：

- 横向宽度按一个周期或一个极对切片来定
- 纵向高度给得稍微宽松一些

原则是：

- 不要太小导致边界影响太强
- 也不要大得离谱拖慢求解

## 6. 第二步：画背铁区域

先画背铁通常最稳。

建议：

- 先画成一个矩形
- 厚度由 `backiron_thickness_mm` 驱动
- 宽度覆盖当前周期区域

如果你的第一版只是单背铁等效，那完全可以接受。

### 6.1 第一版建议

先别追求复杂背铁分层。

只要：

- 它是一个稳定对象
- 厚度是变量驱动
- 后面能用来做 `Bmax_BackIron`

就足够。

## 7. 第三步：画磁钢区域

磁钢建议也先用简单矩形或少量分段矩形。

### 7.1 第一版的重点

你最需要保证的是：

- `magnet_thickness_mm` 改动会影响磁钢厚度
- `pole_arc_ratio` 改动会影响磁钢覆盖宽度

### 7.2 不建议一开始做什么

第一版不建议一开始就做：

- 很复杂的磁钢分段
- 很细的斜极
- 太复杂的 Halbach 近似

这些会让第一版变得难调、难验证。

## 8. 第四步：画气隙区域

气隙区域建议明确单独画出来。

原因是：

- `airgap_mm` 是关键扫描变量
- 你后面会经常用它排查模型是否真的在变

### 8.1 一定要确认

改动 `airgap_mm` 后：

- 线圈和磁钢之间的距离必须真的变

如果这一步没做好，后面很多扫参结果都会失真。

## 9. 第五步：画线圈或等效电流区域

对第一版筛选模型，不要求你一开始就严格复现真实扁铜线细节。

更稳妥的做法是：

- 先画一个或几个等效线圈区域
- 保证位置和尺寸由变量控制
- 激励能正确作用到这些区域

### 9.1 线圈区域最关键的两个变量

必须保证下面两个变量真的驱动线圈位置和尺寸：

- `coil_mean_radius_mm`
- `coil_radial_span_mm`

### 9.2 建议怎么检查

手工改：

- `coil_mean_radius_mm`
- `coil_radial_span_mm`

看线圈是否真的移动和变宽。

## 10. 第六步：给对象起清晰名字

虽然当前脚本不直接强依赖对象名，但强烈建议你在第一版就给关键对象起清楚的名字。

建议至少有：

- `BackIron`
- `Magnet`
- `AirGap`
- `CoilA`
- `Region`

如果你用了多个磁钢或多个线圈，也建议有规律命名，比如：

- `Magnet_N`
- `Magnet_S`
- `CoilA_Pos`
- `CoilA_Neg`

这会让后面做报表、做场计算、排错都容易很多。

## 11. 第七步：指定材料

第一版建议按“稳定优先”的策略。

### 11.1 背铁

给 `BackIron` 指定一个稳定钢材。

### 11.2 磁钢

给磁钢指定一个稳定永磁材料。

### 11.3 线圈

给线圈或导体区域指定铜或等效材料。

### 11.4 外围区域

全部设成 `Air`。

## 12. 第八步：处理边界和周期

第一版你有两种路线。

### 路线 A：简单外空气区

优点：

- 最容易先跑通

缺点：

- 物理等效没那么紧

### 路线 B：周期边界

优点：

- 更接近周期切片思路

缺点：

- 第一版更容易在边界设置上花太多时间

如果你还在第一轮模板搭建阶段，我更建议你先用路线 A 跑通，再逐渐升级到路线 B。

## 13. 第九步：把激励接上

这是模型是否“有用”的关键。

### 13.1 第一版建议

先保证下面两个工况是可表达的：

- 有电流的 loaded 工况
- 零电流的 cogging 工况

### 13.2 你至少要确认的事

变量：

- `phase_current_rms`

必须真的能影响激励。

最简单的检查方法是：

1. 把 `phase_current_rms` 设成 `3A`
2. 解一次
3. 再把它设成 `0`
4. 再解一次
5. 看 torque 是否真的变化

如果两次结果完全一样，说明激励没有真正连上变量。

## 14. 第十步：创建 `Setup_2D`

第一版建议：

- 稳定优先
- 不要过细网格
- 不要先开很重的数据保存

你的目标不是做一份论文级单案例，而是让它能后续连续跑几十个 DOE case。

## 15. 第十一步：先手工跑一次

在建报表前，建议你先完成一次手工求解。

检查这些内容：

- 模型能顺利求解
- 不会明显报材料、边界、激励错误
- 模型不会因为某个变量表达式错误而崩掉

## 16. 第十二步：再去建 5 个报表

报表部分建议你直接配合下面这份文档一起做：

- `templates/AEDT_5_REPORTS_SETUP_GUIDE.md`

因为模型和报表最好分成两个步骤处理：

- 先确保模型能算
- 再确保报表能导

## 17. 第十三步：建完后做一次自动复核

模型和报表都完成以后，建议跑两类脚本。

### 17.1 收口脚本

- `scripts/bootstrap_linear2d_template.py`

用途：

- 补变量
- 补 helper 变量
- 检查 setup
- 保存标准模板路径

### 17.2 复核脚本

- `scripts/validate_linear2d_template.py`

用途：

- 查设计名
- 查 setup
- 查变量
- 查报表
- 查报表导出能力

## 18. 第十四步：判断你的 2D 模型是否达到“可交给 agent”的标准

满足下面这些，就基本可以开始第一轮托管粗筛：

- 设计名是 `Linearized2D`
- `Setup_2D` 存在
- 必需变量存在
- helper 变量存在
- 5 个报表存在
- 5 个报表能导出 CSV
- 模板已经保存到：
  - `templates/linear2d_template.aedt`

## 19. 如果你现在只想知道“我下一步最该做什么”

最推荐的顺序是：

1. 把 2D 模型本体按这份文档建完整
2. 按 `AEDT_5_REPORTS_SETUP_GUIDE.md` 建 5 个报表
3. 跑：
   - `scripts/validate_linear2d_template.py`
4. 如果复核通过，再跑：
   - `.\launchers\Queue-2DScreening.ps1`
