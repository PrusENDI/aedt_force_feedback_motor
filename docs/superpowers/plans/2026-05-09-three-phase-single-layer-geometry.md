# Milestone 4 Three-Phase Single-Layer Geometry Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a pure-2D Milestone 4 diagnostic stage that derives Phase A/B/C logical copper from the validated V3.5 Phase A full-layer source, calculates same-plane overlap/clearance evidence, and writes V5-ready JSON/Markdown/SVG handoff artifacts.

**Architecture:** Keep V3.5 Phase A as the canonical geometry source and add a separate V4 module around it. Phase B/C are generated by angularly transforming Phase A `full_layer_regions[]`; no phase may be reinterpreted into a new geometry language. V4 pass/fail means the diagnostic and V5 handoff pipeline executed correctly, not that same-plane three-phase copper is physically feasible. V4 remains pure Python and 2D-first, while preserving `[x, y, 0.0]` point records so V5 six-layer Z stacking can extend the same data contract without rewriting geometry.

**Tech Stack:** Python 3.10 in the PyAEDT environment, `unittest`, Shapely, existing V3.5 geometry helpers, plain SVG output.

## Pass Semantics

V4 intentionally separates execution success from same-plane physical feasibility.

Markdown and JSON must calculate:

```python
v4_passed = (source_passed is True) and (overlap_area_calculated is True)
```

Same-plane phase overlap is expected for the dense V3.5 default geometry and must not make `v4_passed` false. Same-plane overlap and clearance violations are diagnostic evidence for V5, recorded under `same_plane_issues` and `phase_pair_metrics`.

Expected default status:

```python
{
    "source_passed": True,
    "overlap_area_calculated": True,
    "same_plane_feasibility_passed": False,
    "v4_passed": True,
    "v5_spatial_handoff_ready": True,
    "blocking_issues": [],
    "same_plane_issues": [
        "phase_pair_overlap_detected:A_B",
        "phase_pair_overlap_detected:B_C",
        "phase_pair_overlap_detected:C_A",
    ],
}
```

`blocking_issues` are reserved for broken inputs, broken transforms, missing metrics, invalid angle conversion, lost grouped geometry, radial-wave regression, or incomplete V5 handoff structure.

## Algorithm

1. Build or load the V3.5 Phase A source using `dxf_copper_phase_chain.build_phase_chain_geometry()`.
2. Fail fast unless `v35_full_layer_passed is True` and the source has grouped `full_layer_regions[]`.
3. Normalize V4 spec and derive `mechanical_phase_offsets_deg` from electrical `phase_offsets_deg / pole_pairs_count`.
4. For each phase in `A/B/C`, rotate Phase A geometry by the phase's mechanical rotation angle:
   - `outline_points_xy_mm`
   - `aedt_polyline_points_mm`, preserving z as `0.0`
   - `centerline_points_xy_mm`
   - terminal pad centers
   - terminal keepout centers
5. Emit both XY and XYZ grouped point fields:
   - XY is authoritative for Shapely validation.
   - XYZ is authoritative for downstream AEDT/V5 handoff shape, with z fixed at `0.0` in V4.
6. Rewrite identity metadata deterministically:
   - `phase`
   - `region_id`
   - `segment_id`
   - `source_phase`
   - `source_region_id`
   - `source_segment_id`
   - `electrical_offset_deg`
   - `mechanical_rotation_deg`
7. Build Shapely polygons for all regions without merging them into a single copper body.
8. Validate per-phase internal geometry:
   - grouped region count matches Phase A
   - region point counts are preserved by transform
   - self/internal overlap remains absent
   - radial-wave score remains valid
   - bounding diameter remains within source limits
9. Validate phase-to-phase geometry:
   - `phase_to_phase_minimum_clearance_mm`
   - `phase_pair_overlap_area_mm2`
   - pairwise overlap issue codes for `A_B`, `B_C`, `C_A` recorded as `same_plane_issues`
10. Validate terminal keepouts as feasibility checks only:
    - keepout circle vs opposite-phase copper overlap
    - keepout circle vs same/other terminal keepouts
    - no final terminal escape geometry is generated
11. Build `v5_spatial_handoff` with reserved layer-instance records for `A-B-C-C-B-A`; V4 does not assign final z spacing.
12. Set `overlap_area_calculated=True` only after all phase-pair overlap areas are numeric.
13. Set `v4_passed = source_passed and overlap_area_calculated`.
14. Generate V4 JSON, Markdown, and SVG preview even when `same_plane_feasibility_passed=False`.

## Validation Gates

V4 execution passes only when:

- V3.5 source passes.
- `pole_pairs_count` is valid.
- `phase_offsets_deg` are treated as electrical degrees.
- `mechanical_phase_offsets_deg` exactly match `phase_offsets_deg / pole_pairs_count`.
- A/B/C all have the same grouped region count as Phase A.
- B/C transformed point counts match the corresponding Phase A point counts.
- No single merged full-layer polygon field is emitted.
- Phase-pair overlap areas were successfully calculated for `A_B`, `B_C`, and `C_A`.
- `overlap_area_calculated == True`.
- `source_passed == True`.
- `v4_passed == True`.
- A/B/C all remain radial-wave valid.
- `v5_spatial_handoff_ready == True`.
- `blocking_issues == []`.

Same-plane physical feasibility is a separate diagnostic gate:

- `same_plane_feasibility_passed == False` is acceptable and expected for the dense V3.5 default geometry.
- Positive phase-to-phase overlap area is recorded in `same_plane_issues`.
- Clearance below `minimum_phase_to_phase_clearance_mm` is recorded in `same_plane_issues`.
- Terminal keepout conflicts are recorded in `terminal_keepout_conflicts` and summarized in the Markdown report.
